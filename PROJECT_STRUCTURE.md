## 📋 File Descriptions

### 🔧 Configuration Files


**Purpose:** Web application configuration
**Contains:**

### 💾 Backend Java Files

#### Model Layer (Data Classes)

**`Event.java`**
- Represents an event in the system
- Contains: id, name, description, date, location, tickets, price
- Uses: Encapsulation (private fields, public getters/setters)

**`Booking.java`**
- Represents a ticket booking
- Contains: id, eventId, customer info, tickets, amount, status
- Uses: Encapsulation

#### DAO Layer (Database Operations)

**`EventDAO.java` (Interface)**
- Defines what operations are available for events
- Methods: getAllEvents(), getEventById(), addEvent(), etc.
- Uses: Abstraction

**`EventDAOImpl.java` (Implementation)**
- Implements the EventDAO interface
- Contains actual SQL queries
- Handles database connections
- Uses: Polymorphism, Abstraction

**`BookingDAO.java` (Interface)**
- Defines booking operations
- Uses: Abstraction

**`BookingDAOImpl.java` (Implementation)**
- Implements booking database operations
- Uses: Polymorphism, Abstraction

#### Controller Layer (Servlets)

**`BaseServlet.java`**
- Parent class for all servlets
- Common methods: sendJsonResponse(), parseJsonRequest()
- Uses: Inheritance (other servlets extend this)

**`EventServlet.java`**
- Handles HTTP requests for events
- Endpoints: GET, POST, PUT, DELETE for /api/events
- Uses: Inheritance (extends BaseServlet), Polymorphism

**`BookingServlet.java`**
- Handles HTTP requests for bookings
- Endpoints: GET, POST, PUT, DELETE for /api/bookings
- Uses: Inheritance, Polymorphism

**`LocalDateTimeAdapter.java`**
- Converts LocalDateTime to/from JSON
- Used by Gson for JSON serialization

#### Utility Layer

**`DatabaseUtil.java`**
- Manages database connections
- Contains connection settings
- Provides getConnection() method

**`CorsFilter.java`**
- Enables Cross-Origin Resource Sharing
- Allows frontend to call backend APIs

## 🔄 How Files Work Together

### Example: Booking a Ticket

```
1. User clicks "Book Now" (index.html)
   ↓
2. JavaScript opens modal (app.js → openBookingModal())
   ↓
3. User fills form and submits (index.html)
   ↓
4. JavaScript sends POST request (app.js → fetch API)
   ↓
5. Request hits BookingServlet (BookingServlet.java → doPost())
   ↓
6. Servlet uses BookingDAO (BookingDAOImpl.java → createBooking())
   ↓
7. DAO executes SQL (DatabaseUtil.java → getConnection())
   ↓
8. Database stores booking (MySQL)
   ↓
9. Response sent back (JSON)
   ↓
10. JavaScript shows success message (app.js)
```

### Example: Viewing Events

```
1. Page loads (index.html)
   ↓
2. JavaScript runs (app.js → loadEvents())
   ↓
3. GET request to /api/events/ (fetch API)
   ↓
4. EventServlet handles request (EventServlet.java → doGet())
   ↓
5. Servlet calls DAO (EventDAOImpl.java → getAllEvents())
   ↓
6. DAO queries database (SQL SELECT)
   ↓
7. Results converted to Event objects (Event.java)
   ↓
8. Servlet sends JSON response (BaseServlet.java → sendJsonResponse())
   ↓
9. JavaScript receives data (app.js)
   ↓
10. Events displayed on page (DOM manipulation)
```

## 🎯 Layer Responsibilities

### Model Layer
**Files:** `Event.java`, `Booking.java`
**Job:** Define data structure
**Knows about:** Nothing (just data)

### DAO Layer
**Files:** `*DAO.java`, `*DAOImpl.java`
**Job:** Database operations
**Knows about:** Models, Database

### Controller Layer
**Files:** `*Servlet.java`
**Job:** Handle HTTP requests
**Knows about:** Models, DAOs, HTTP

### Utility Layer
**Files:** `DatabaseUtil.java`, `CorsFilter.java`
**Job:** Support other layers
**Knows about:** Specific tasks

### View Layer
**Files:** `index.html`, `style.css`, `app.js`
**Job:** User interface
**Knows about:** API endpoints

## 📦 Build Output

Created when you run `mvn package`

**Contains:**

- `event-management-system.war` - Deployable web archive

**WAR file structure:**
